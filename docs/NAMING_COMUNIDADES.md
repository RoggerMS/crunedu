# Naming funcional — Comunidades (iteración actual)

## Decisión de naming único

- Término oficial para producto/UI en esta iteración: **"Comunidades"**.
- No se debe usar terminología mezclada en texto visible como "grupos" o "clubs".
- "Clubs" queda reservado como **experimento futuro** y fuera del alcance actual.

## Inventario de texto visible revisado

### 1) Navegación

- Menú principal: `Comunidades` (`/app/comunidades`).
- CTA lateral/rápida: `Unirme a una comunidad`.

### 2) Títulos de páginas

- Página principal de comunidades: `Comunidades`.
- Flujo relacionado mantiene referencia consistente a comunidad/comunidades.

### 3) Empty states

- Feed principal: `No hay publicaciones aún` + referencia a "tu comunidad".
- Comunidades: estados de carga/error con "comunidades".

### 4) Botones y acciones

- `Elegir comunidades`.
- `Unirme a una comunidad`.
- Selectores: `Selecciona una comunidad`.

## Resultado de consistencia

- En el texto visible de la aplicación se mantiene el término **comunidad/comunidades** de forma consistente.
- No se introducen cambios de rutas ni de modelos técnicos por esta decisión de naming.

## Regla para próximas iteraciones

- Mantener **Comunidades** como término único de UX hasta que exista una definición formal del experimento "clubs" (objetivo, hipótesis, segmentación y métricas).
